## Systems

### 1 · Editor timeline (timeline.js + /api/timeline/*)
- `GET /api/timeline/runs` (run picker) · `GET /api/timeline/manifest?run=&video=` · `POST /api/timeline/render`.
- Two run types: **EDL runs** (cut_edl.json + broll_edl.json + master.srt → A-CAM/DIALOG/B-ROLL/CAPTIONS tracks) and **sidecar runs** (`<video>.timeline.json` from variant_factory / assemble.py / streamlined_ad → concat B-ROLL + VO + CAPTIONS). Both get a TITLES track (titles.json).
- **Tools:** V select (move/trim; v1 drag = reorder w/ ripple) · C razor (click a clip to split) · T text (click monitor — letterbox-aware — or timeline to place a title; inspector edits text/size/color). Double-click caption/title bars to edit text. Delete key deletes. Drag-drop file rows onto the timeline inserts clips (sidecar v1 = full clip at position; EDL v2 = 3 s cutaway).
- **⟳ re-render** = deterministic: EDL branch (render.py → broll_overlay.py → ASS re-burn → titles chained in one encode) or sidecar branch (`reassemble.py`: concat → mux VO → burn captions → `--titles-ass` pass). Versioned outputs (`*_edit<N>.mp4`), `*.orig` backups on first touch, render-semaphore-gated (max 2).
- **🎬 ask Kino to fix** = drops a human-readable change list into the composer instead.
- **INVARIANT (bit us 3×):** generated artifacts `captions_edit*` / `titles_edit*` must NEVER be discovered as caption sources or style templates. All three discovery sites filter them — keep it that way.
- Sidecar runs are editable ONLY if VO survives on disk (`vo_file` field / `<stem>.vo.*` / `vo/*.mp3`) — variant_factory + streamlined_ad now persist VO + ASS + SRT next to outputs. Pre-2026-07-13 variant runs are read-only (status bar says so).
- **MUSIC + HOOK tracks (2026-07-20):** sidecar manifests now emit a MUSIC audio lane (teal, from the `music` field) and a HOOK band (magenta, from the `hook` field) — both editable on **streamlined runs only** (variants don't record `music`). Inspector: MUSIC has swap (file picker, audio) + "✕ remove music"; HOOK has a text editor (no delete). Edits ride new payload keys `music` (path or **null**=remove) / `hook` (string). **Streamlined runs re-render through a NEW branch that regenerates via `streamlined_ad.py`** (not reassemble.py — it can't do music beds or the hook/bullet/CTA ASS): the branch reads the sidecar (format/bullets/cta/bg/brand/resolution), applies the hook/music/bg edits, writes `<stem>_edit<N>.mp4` in the run folder (no Final Output re-copy). text-vo re-render re-voices via the persisted `voice_id` (falls back to Sean DiH2). Verified end-to-end on a throwaway run (hook edit + music removal → correct output, zero audio streams).
- `_rebuild_ass` balance-wraps cues >24 chars onto two lines (edited sentences used to overflow frame).
